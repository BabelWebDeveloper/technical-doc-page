Wprowadzenie

JavaScript to wieloplatformowy, zorientowany obiektowo język skryptowy. Jest to mały i lekki język. Wewnątrz środowiska hosta (na przykład przeglądarki internetowej) JavaScript może być połączony z obiektami swojego środowiska, aby zapewnić nad nimi kontrolę programową. JavaScript zawiera standardową bibliotekę obiektów, takich jak Array, Date i Math, oraz podstawowy zestaw elementów języka, takich jak operatory, struktury sterujące i instrukcje. Core JavaScript może być rozszerzany do różnych celów poprzez uzupełnianie go dodatkowymi obiektami; na przykład: Client-side JavaScript rozszerza podstawowy język, dostarczając obiekty kontrolujące przeglądarkę i jej obiektowy model dokumentu (DOM). Na przykład rozszerzenia po stronie klienta umożliwiają aplikacji umieszczanie elementów w formularzu HTML i reagowanie na zdarzenia użytkownika, takie jak kliknięcia myszą, wprowadzanie formularza i nawigacja po stronie. Server-side JavaScript rozszerza język podstawowy, dostarczając obiekty związane z uruchamianiem JavaScript na serwerze. Na przykład rozszerzenia po stronie serwera pozwalają aplikacji komunikować się z bazą danych, zapewniać ciągłość informacji z jednego wywołania aplikacji do drugiego lub wykonywać operacje na plikach na serwerze.

JavaScript i Java

JavaScript i Java są pod pewnymi względami podobne, ale pod innymi zasadniczo się różnią. Język JavaScript przypomina Javę, ale nie ma statycznego typowania i silnego sprawdzania typu Javy. JavaScript jest zgodny z większością składni wyrażeń Java, konwencjami nazewnictwa i podstawowymi konstrukcjami przepływu sterowania, co było powodem zmiany nazwy z LiveScript na JavaScript. W przeciwieństwie do systemu klas w czasie kompilacji Java zbudowanych przez deklaracje, JavaScript obsługuje system wykonawczy oparty na małej liczbie typów danych reprezentujących wartości liczbowe, logiczne i łańcuchowe. JavaScript ma model obiektowy oparty na prototypach zamiast bardziej popularnego modelu obiektowego opartego na klasach. Model oparty na prototypie zapewnia dynamiczne dziedziczenie; to znaczy, co jest dziedziczone, może się różnić dla poszczególnych obiektów. JavaScript obsługuje również funkcje bez żadnych specjalnych wymagań deklaratywnych. Funkcje mogą być właściwościami obiektów, wykonywanymi jako metody o luźnym typie. JavaScript jest językiem o bardzo wolnym kształcie w porównaniu z Javą. Nie musisz deklarować wszystkich zmiennych, klas i metod. Nie musisz się martwić, czy metody są publiczne, prywatne czy chronione i nie musisz implementować interfejsów. Zmienne, parametry i typy zwracane przez funkcje nie są jawnie wpisane.

Podstawy

Składnia JavaScripta zapożycza wiele rozwiązań użytych w Javie, jednak w jej konstrukcji można zauważyć również wpływy języków takich jak Awk, Perl i Python JavaScript jest językiem, w którym rozróżnialna jest wielkość liter, oraz wspierany jest standard znaków Unicode. W JavaScripcie instrukcje nazywane są wyrażeniami i rozdzielane średnikiem (;). Spacje, tabulatury i znaki nowej linii zaliczają się do grupy tak zwanych znaków białych. Kod źródłowy skryptów napisanych w JavaScripcie skanowany jest przez interpreter od lewej do prawej i konwertowany w sekwencje elementów wejścia, które reprezentowane są przez tokeny, znaki kontrolne, znaki przerwania linii, komentarze i białe znaki. ECMAScript definiuje również zestaw słów kluczowych i literałów oraz zasady automatycznego umieszczania średników (ASI), tak by zakończyć dane wyrażenie, jednakże wykorzystywanie tej zasady wiąże się z efektami ubocznymi, dlatego zaleca się samodzielne umieszczanie średników. Więcej informacji na ten temat znajdziesz w szczegółowym opisie gramatyki języka.
